03 Core Technology

Four Engines Working in Harmony

Each engine solves a different dimension of the mapping problem. Together, they achieve what no single approach can.

NS
Name Similarity Engine
Custom segmentation model + abbreviation expansion classifier understands that custId = customer_id and empFirstNm = employee_first_name. Detects directional conflicts like minPrice vs maxPrice.
VS
Value Similarity Engine
Statistical profiling with 10+ signals (MCV cosine, histogram intersection, CDF similarity, entropy, containment) to compare actual data distributions. Catches mismatches names alone would miss.
JG
Join Graph Builder
Automatically discovers foreign key relationships across tables using value overlap, constraint compatibility, and name patterns. Builds a weighted join graph with Dijkstra-based path planning.
TF
Transform Classifier
Multi-class predictor with 93+ transformation types trained on 46,000+ examples. Handles everything from simple renames to complex arithmetic, date operations, and multi-hop lookups.

05 Resilient Architecture

Three-Tier Candidate Generation
with Intelligent Fallback

If the LLM is uncertain, the system automatically falls back to ML-based retrieval, then to rule-based heuristics. Zero single points of failure.

V3
Tier 1: LLM Schema Matcher
Fine-tuned Phi-3-mini understands schema semantics, FK relationships, and compositional reasoning. Outputs structured mappings with confidence. Handles synonyms, arithmetic, multi-hop joins.
V2
Tier 2: Bi-Encoder + Cross-Encoder
SentenceTransformer bi-encoder for fast retrieval across thousands of candidates. Cross-encoder reranks top-K for precision. Trained on serialized schema text.
V1
Tier 3: Algorithmic Engine
ANN indexing + beam search over the join graph. Composite candidate generation with feasibility scoring. Always available as the baseline guarantee.
06 Innovation Spotlight

Name Similarity That Actually Works

Most tools fail on abbreviated or concatenated column names. Our 4-stage pipeline handles real-world naming chaos.

Stage 1: Regex Split
empFirstNm → [emp, First, Nm]
Stage 2: ML Segmentation
Conv1D + Transformer + BiLSTM per-character boundary detection
Stage 3: Expansion
emp → employeeNm → nameDict + Classifier hybrid
Stage 4: Similarity
Embedding cosine + conflict detection (min vs max, buyer vs seller)
cfgMgr → configuration_manager     txnAmt → transaction_amount     acctBal → account_balance
