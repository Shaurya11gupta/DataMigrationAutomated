User steps Flask API Engines & discovery LLM model
STEP 1
Source Ingestion
User defines source tables (JSON or visual builder). On submit, JSON is converted to Table objects via json_to_tables.
JOIN DISCOVERY
JoinGraphBuilderV2
Uses ValueSimilarity + ConstraintCompatibilityJoin to discover FK relationships between tables. Outputs List[JoinEdge] with confidence scores.
STEP 2 & 3
Visualization & Target
Join graph is displayed. User defines target schema (tables, columns with descriptions).
CANDIDATE SELECTION
SchemaMatcherLLM (V3)
Phi-3-mini + LoRA adapter (schema_matcher_llm1). For each target column: builds prompt (schema + joins + target), generates mapping, parses source_columns and transform_type.
DataMigrationCrane — Run app | pipeline_bridge + candidate_generation_v3
